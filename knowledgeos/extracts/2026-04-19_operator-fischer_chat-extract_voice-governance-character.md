## 2) New Core Principles Anchored

### 2.1 Governance / Architecture
- Agentic AI does **not** replace iPaaS in the German legal/compliance context.
- iPaaS becomes a **control layer**, not an obsolete transition technology.
- Strong architecture pattern:
  - **Agentic AI = bounded reasoning layer**
  - **iPaaS = governed execution layer**
- Key operating idea:
  - not maximal autonomy
  - but **trust-capable effectiveness**

### 2.2 Voice / Text Interaction
- Voice is an **interface layer**, not the operating core.
- Voice is for:
  - coaching
  - drill
  - roleplay
  - decision acceleration
  - fast interaction
- Text is for:
  - context loading
  - precision
  - stable structure
  - result capture
- Canonical formula:
  - **Text loads context**
  - **Voice creates flow**
  - **Text secures output**

### 2.3 Character Development
- Character is treated as a **leadership discipline**, not a vague personality topic.
- Target is not stronger expression alone, but:
  - **more precisely led strength**
- Risks explicitly identified:
  - overengineering
  - social over-density
  - meta-expansion
  - weak outcome measurement

### 2.4 Model Usage / LLM Routing
- Benchmarks alone do not determine operational dominance.
- **Routing > model religion**
- **Harness > hype**
- Use models according to task class, not brand loyalty.

## 5) Voice-First Workflow — Condensed Extract

### Operational thesis
Voice is strongest when used as a **high-speed interaction layer** within a hybrid system.

### Canonical loop
1. load context in text
2. set role clearly
3. run voice interaction
4. summarize back into text
5. derive next action / to-do

### Good use cases for voice
- role-based coaching
- rhetorical drills
- language training
- decision rehearsal
- reflective dialogue
- customer/sales simulations

### Bad use cases for voice alone
- high precision architecture
- legal sensitive wording
- complex deep work without text anchoring
- stable documentation needs
