# Raw Input → Artifact Example

## Raw Input

"We have problems in our cold chain logistics. Things are too slow, errors happen, and I don’t know how to fix it."

---

## Step 1 — Problem Class

- Type: operational process problem
- Domain: cold chain logistics
- Nature: performance + error rate + lack of structure

---

## Step 2 — Mode Selection

- Primary: BUILD
- Secondary: AUDIT

---

## Step 3 — Structured Output

### Problem Breakdown

- Throughput inefficiency
- Error sources in picking / handling
- Lack of standardized process logic

### Hypotheses

- Suboptimal warehouse layout
- Missing KPI tracking
- Weak process enforcement

### Initial Framework

- Input: goods, orders, temperature constraints
- Process: receiving → storage → picking → dispatch
- Output: correct, on-time, temperature-safe delivery

---

## Step 4 — Artifact

**Cold Chain Process Audit Framework**

1. KPI Layer
   - picks per hour
   - error rate
   - temperature deviations

2. Process Layer
   - inbound control
   - storage logic
   - picking method
   - outbound validation

3. Risk Layer
   - temperature breaks
   - mispicks
   - delays

---

## Step 5 — Governance Check

- Assumptions: basic process exists
- Risks: missing data, incorrect KPI definition
- Classification: APPROVED WITH WARNING

---

## Result

Raw input has been transformed into:
- structured problem understanding
- initial system framework
- reusable audit logic

---

## GosseOS Principle

Messy input is not a problem.
Unstructured output is.
