# Public Short Description

## Brutally clear version

**GosseOS is the working system. KnowledgeOS is the memory system.**

GosseOS helps turn raw input into:
- the right problem class
- the right mode
- the right artifact
- the right level of control

KnowledgeOS preserves what should survive the session:
- core principles
- reusable prompts
- frameworks
- extracts
- canon candidates

In plain language:

**This project is about using AI as a structured work system instead of a random chat tool.**

Not just:
- prompt → answer

But:
- raw input → problem class → mode → artifact → verification → reusable knowledge

That is the whole shift.

## Slightly expanded version

Most people use AI to get outputs.

This project is about something more useful:
**turning messy ideas, notes, workflows, decisions, and problems into structured, reusable, governance-aware work.**

### GosseOS
GosseOS is the operational layer.
It helps answer:
- What kind of problem is this?
- Which working mode is needed?
- What should be built?
- Where do risk, control, and governance matter?

### KnowledgeOS
KnowledgeOS is the extraction and memory layer.
It helps answer:
- What from this session is worth keeping?
- What is reusable?
- What is only working material?
- What could become canon later?

## One-line version

**GosseOS structures the work. KnowledgeOS preserves and evolves the intelligence behind it.**
