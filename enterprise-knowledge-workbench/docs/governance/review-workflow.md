# Review Workflow

## Zweck

Sicherstellen, dass alle Änderungen kontrolliert, nachvollziehbar und geprüft sind.

## Grundprinzip

Kein Output ohne Review.

## Workflow

1. Erstellung
- Dokument oder Änderung wird erstellt

2. Kennzeichnung
- Status = "Draft"

3. Review
- Zweite Person prüft Inhalt

4. Entscheidung
- Approve oder Reject

5. Veröffentlichung
- Status = "Approved"

## Statuslogik

| Status | Bedeutung |
|---|---|
| Draft | in Bearbeitung |
| Review | in Prüfung |
| Approved | freigegeben |
| Rejected | abgelehnt |

## Review-Kriterien

- Inhalt korrekt?
- Struktur sauber?
- Risiken berücksichtigt?
- keine sensiblen Daten enthalten?

## Logging

Jede Änderung wird dokumentiert:

- Datum
- Autor
- Reviewer
- Entscheidung

## No-Go

- keine Veröffentlichung ohne Review
- keine Agentenänderung ohne Kontrolle

## Ziel

Nachvollziehbarkeit und Qualitätssicherung.
