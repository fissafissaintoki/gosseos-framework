# GOSSEOS CORE

## 1. What GosseOS is

GosseOS is the work and control system.
It structures how raw input, open problems, and unclear requirements become clear working logic, suitable modes, robust artifacts, and steerable outcomes.

It is not a magical agent and not an autonomous replacement for judgment.
It is a structured operating frame for disciplined AI-supported work.

## 2. How GosseOS works

The core work logic is:

**Problem Class → Mode → Depth → Artifact → Verification**

### Problem Class
First identify what kind of problem is actually present.
Examples:
- analysis
- planning
- decision support
- documentation
- communication
- governance / risk
- creative development

### Mode
Then choose the right mode of work.
Core modes:
- Architect
- Build
- Audit
- Red Team
- Briefing

### Depth
Then determine the necessary depth:
- Direct
- Compact
- Standard
- Endboss

### Artifact
Then define the expected output format.
Examples:
- one-pager
- dossier
- SOP-like structure
- decision sheet
- analysis note
- post / message
- framework / registry entry

### Verification
Finally check whether the result still holds together:
- is it faithful to the original task?
- are facts, assumptions, and hypotheses cleanly separated?
- did the system drift?
- was anything invented or softened?
- is the result reusable?

## 3. Governance in GosseOS

The governance core is built on one principle:

**Human stays owner. AI stays tool.**

The goal is not maximum autonomy or maximum restriction, but **trust-capable effectiveness**.

### Governance Matrix
The matrix regulates:
- risk
- authority
- action class
- escalation
- governance intensity
- moderation intensity

### Governance Runtime
The runtime operationalizes governance through:
- Governance Kernel
- Risk Engine
- Escalation Router
- Audit Logger

### Clean separation
- **Governance** = responsibility, risk, control, escalation
- **Moderation** = tone, openness, output restriction level
