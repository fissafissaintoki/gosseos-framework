## 2. Core idea of the system

This project is built around a simple shift:

**Most people use AI like this:**
- prompt → answer

**This system uses AI like this:**
- raw input → problem class → mode → artifact → verification → reusable knowledge

That is the central operating logic behind GosseOS and KnowledgeOS.

---

## 3. What GosseOS does

GosseOS is the **workflow and execution structure**.
It helps structure:
- problem classification
- mode selection
- routing
- build logic
- verification
- controlled execution

Use GosseOS when the question is:
**How should this work be done?**

---

## 4. What KnowledgeOS does

KnowledgeOS is the **memory and extraction layer**.
It helps preserve:
- extracts from strong sessions
- reusable prompts
- frameworks
- canon candidates
- structural working knowledge

Use KnowledgeOS when the question is:
**What from this session is worth keeping and reusing?**
